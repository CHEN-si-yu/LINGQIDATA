## 1. 当前阶段结论

- 项目已进入**第二阶段"批量因子产出"**。
- 核心框架完整且稳定：数据加载层、因子注册表、自动模块加载、构建引擎、输出存储、IC分析。
- 截至 `2026-05-12`，已实现 **101 个因子**（97 个特征因子 + 4 个 target 标签），覆盖所有 8 个类别。
- 全部因子已通过构建验证和初步 IC 分析，截面信号有效。
- 当前的第一要务是**行业中性化 + 增强因子 + 因子筛选**，提升因子库实战价值。
- 数据环境与旧项目（`lingqingData - 副本/featureengineering`）有本质差异：存储格式从 `.fea` 变为 `.parquet`，日期和代码格式变化，事件表拆分，行业映射来源变化。

## 2. 项目目标

### 2.1 设计目标

1. 把原始 `.parquet` 数据统一封装成可复用的数据访问层（`dataset.py`）。
2. 把"新增一个因子"约束成低成本动作——只新增因子函数，不改主流程。
3. 优先批量沉淀传统的、统计意义明确的基础因子。
4. 让单因子输出（`.fea` + `manifest.json`）标准化。
5. 让同一套输入源支持基础单因子、增强因子、组合因子持续迭代。

### 2.2 当前研发原则

- 第一阶段以"多产出基础因子"为优先目标。
- 因子是否值得先做，首要看它是否有明确经济含义、统计含义或经典研究背景，而不是先看短期样本内表现。
- 不把高 IC 作为立项门槛。
- 先把价量、估值、质量、事件、资金流这些传统因子族铺全，形成可复用的因子库。
- **所有因子必须使用历史数据生成，严禁引入未来信息。** 在日期 T 构建因子时，只能使用 T 日及之前可获得的收盘数据。例如：T 日的 `mom_20` 用 `close[T] / close[T-20] - 1`，不涉及 T+1 及之后的数据。标签类因子（如 `label_ret`）天然包含未来信息，仅用于模型训练目标，不作为选股因子使用。

### 3.1 关键模块职责

- **`settings.py`**：负责项目根目录、输入数据根目录（`data/`）、输出目录的统一配置。支持环境变量 `FEATURE_ENGINEERING_PROJECT_ROOT` 和 `FEATURE_ENGINEERING_SOURCE_ROOT` 覆盖。

- **`dataset.py`**：读取 `.parquet` 文件，将 `(trade_date, stock_code)` 统一规整为 MultiIndex `(Date, Code)`。`Date` 为 `YYYYMMDD` 字符串，`Code` 保留交易所后缀（如 `000001.SZ`）。提供 `load_panel()`（日频面板）、`load_financial_panel()`（财报→日频前向填充）、`load_stock_pool()`（上市股票池）、`load_industry_map()`（个股→行业映射）。

- **`registry.py`**：`@register_factor` 装饰器驱动的因子注册系统。每个因子注册 `name`、`description`、`category`、`thesis`（经济假设）、`dependencies`（依赖数据路径）。

- **`factor_loader.py`**：使用 `pkgutil.walk_packages` 递归扫描 `factors/` 目录树下所有模块并 import，支持子包（如 `target/`），无需手工维护总表。

- **`ic_analysis.py`**：IC / ICIR 分析模块。`compute_ic(factor, target)` 按日期计算截面 rank IC（Spearman）或 Pearson IC，输出 `ICResult`（IC mean、IC std、ICIR、IC>0 ratio、t-stat）。`compute_ic_matrix(factors, targets)` 批量评估多因子 × 多标签。`get_ic_series(factor, target)` 返回 IC 时间序列。

- **`builder.py`**：负责单因子构建 `build_factor()`、顺序批量 `build_many()`、并行批量 `build_many_parallel()`。总入口 `build_all()`。

- **`storage.py`**：校验输出格式（单列、MultiIndex），写出 `.fea` 和 `manifest.json`。

- **`utils.py`**：`safe_divide`、`cross_sectional_rank`、`rolling_group_mean/std/sum/max/min/skew/kurt`、`rolling_corr_by_group`。

- **`cli.py`**：命令行参数 `--list` / `--all` / `--factor` / `--jobs` / `--sequential`。

## 5. 数据访问约定

### 5.1 日频面板数据

以下文件通过 `DataRepository.load_panel()` 加载：

- `daily.parquet` / `daily_adj.parquet`
- `finance.parquet`
- `main_fund_flow.parquet` / `main_fund_flow_overview.parquet`
- `margin_detail.parquet`
- `limit_up.parquet` / `limit_list.parquet` / `top_list.parquet`
- `dragon_tiger.parquet`（需聚合后使用）
- 周线/月线 K 线（结构与日线一致，可直接加载）

加载后统一规整为：

- MultiIndex：`Date`、`Code`
- `Date`：`YYYYMMDD` 字符串（去 `-`）
- `Code`：保留交易所后缀的原生格式（如 `000001.SZ`）
- 辅助列（`stock_name`、`name`）自动剔除

### 5.2 财报频率数据

以下文件通过 `DataRepository.load_financial_panel()` 加载：

- 财务三表：`financial_indicator.parquet` / `balancesheet.parquet` / `income.parquet` / `cashflow.parquet`
- 股东户数：`holder_number.parquet`（`ann_date` 驱动）
- 质押统计：`pledge_stat.parquet`（`end_date` 驱动，月度频率）

加载逻辑：

1. 读取原始报表数据
2. 以 `ann_date`（披露日）为时间锚点
3. 按个股将报告期数据前向填充到所有交易日
4. 输出格式与日频面板一致：MultiIndex `(Date, Code)`

**注意**：财报数据不是现成的 TTM，而是"日频前向填充的最新累计财报值"。因此像 `cash_earnings_quality` 这类因子，必须先按 `end_type` 从累计值还原 TTM 再计算。

### 5.3 股票池

`DataRepository.load_stock_pool()`：

- 来源：`stock_list.parquet`
- 过滤：`list_status == 'L'`（仅上市状态）
- 返回：`Code`、`industry`、`area` 三列
- 当前上市股票数：约 5500 只

### 5.4 行业映射

`DataRepository.load_industry_map()`：

- 返回 `Series`：`Code` → `industry`
- 约 111 个行业分类
- 用于行业中性化、行业内部排名、行业相对因子

## 6. 输入源详细说明

这一节是最重要的"可调用数据地图"。每个字段标注了 dtype 和空值率，方便快速判断可用性。

> **通用约定**：
> - `stock_code`：带交易所后缀的原生代码，如 `000001.SZ`、`600000.SH`，约 5,700+ 个唯一值

> - `stock_name` / `name`：中文名称，加载时被 `dataset.py` 自动剔除
> - 空值率 < 5% 的字段可直接使用；空值率 > 20% 的字段需谨慎评估覆盖率

## 8. 输出规范

每个因子输出两个文件：

- `data/factors/<factor_name>.fea`
- `data/manifests/<factor_name>.json`

**因子文件要求**：

- 只能有 1 列，列名等于因子名
- 索引必须是 MultiIndex `(Code, Date)` 或 `(Date, Code)`
- 写出前按 `Date`、`Code` 排序

**manifest 记录**：

- 因子名、描述、分类、经济假设、依赖数据
- 行数、非空行数、覆盖率
- 构建时间

## 9. 运行方式

### 9.1 命令行

```bash
cd featureengineering

# 列出所有已注册因子
python build_factors.py --list

# 构建全部因子
python build_factors.py --all

# 构建单个因子
python build_factors.py --factor mom_20 --sequential

# 指定数据源路径
python build_factors.py --all --source-root E:/BaiduSyncdisk/lingqiData/data

# 并行构建
python build_factors.py --all --jobs 4
```

### 9.2 PyCharm 直接运行

可直接运行 `build_factors.py`，默认参数由文件顶部常量控制：

- `RUN_MODE`：`"all"` / `"list"` / `"custom"`
- `FACTORS`：当 `RUN_MODE == "custom"` 时指定因子名列表
- `JOBS`：并行数
- `SEQUENTIAL`：是否顺序执行
- `SOURCE_ROOT`：自定义数据源路径

### 10.8 Target 类（4 个，目标标签，不作为选股因子）

统一交易逻辑：**T 日收盘后计算因子 → T+1 开盘买入 → 持有 N 日 → T+1+N 开盘卖出**。公式：`open[t+1+N] / open[t+1] - 1`。

| 因子名 | 持有期 | 公式 | 覆盖率 |
|---|---|---|---|
| `label_ret_1d` | 1 日 | `open[t+2] / open[t+1] - 1` | ~99.9% |
| `label_ret_5d` | 5 日 | `open[t+6] / open[t+1] - 1` | ~99.6% |
| `label_ret_10d` | 10 日 | `open[t+11] / open[t+1] - 1` | ~99.3% |
| `label_ret_20d` | 20 日 | `open[t+21] / open[t+1] - 1` | ~98.6% |

> **注意**：Target 类因子天然包含未来信息（前视偏差），仅用于模型训练监督目标，严禁作为选股特征因子使用。

## 11. 后续开发方向

### 11.1 行业中性化（最高优先级）✅ 新增

基于 `stock_list.parquet` 的 `industry` 字段（111 行业），对核心因子做行业内部排名，消除行业偏差：

- 对 IC 显著的因子（`bp`、`ep_ttm`、`mom_20`、`price_volume_corr_20`、`limit_up_consecutive`）产出行业中性化版本
- 放入 `enhanced.py` 或新增 `factors/neutral.py`

### 11.2 细分因子补齐（中优先级）

- **Price**：`breakout_60`（突破因子）、ATR（真实波幅）、`pct_chg_vol_20`（涨跌幅波动率）
- **Valuation**：`dv_yield_ttm` 与 `dv_ratio` 的融合版本
- **Fund Flow**：基于 `main_fund_flow_overview.parquet` 的 `net_amount_rate` 系列（历史较短但字段更便捷）
- **Event**：`dragon_tiger.parquet` 席位级别聚合因子（席位数量、机构席位占比）
- **筹码分布**：需要先完成 `cyq_chips/` 的聚合工作，再产出筹码集中度/获利盘比例等因子

### 11.3 因子筛选与评估

- 对全量 97 个特征因子做系统的 IC 衰减分析（IC decay across horizons）
- 因子相关性矩阵 → 剔除冗余因子（如 `mom_20` 与 `mom_20_vol_adj` 相关性预计>0.9）
- 分层回测（按因子值分 5/10 组，计算多空收益）
- 设定 ICIR 阈值（如 |ICIR| > 0.3）筛选核心因子池

### 11.4 增强模块扩充

- 去极值（MAD / 百分位截尾）
- 市值中性化
- 多窗口融合（如 `mom_5_10_20_60` 的等权/ICIR 加权合成）
- 因子时效性分析（半衰期估计）

### 11.5 可选：财务 TTM 还原

`income.parquet` 和 `cashflow.parquet` 的累计值需按 `end_type` 还原 TTM，当前 `financial_indicator.parquet` 已有大部分常用 TTM 字段，优先级较低。

## 13. 给后来接手者的最短建议

1. 先看 `src/featureengineering/factors/price.py` 中的 `mom_20`，它是标准模板。
2. 再看本文件第 6 节，确认你要用哪份底层数据。
3. 优先复用 `utils.py` 中的 `cross_sectional_rank()`、`rolling_group_*` 等公共逻辑。
4. 新因子放到合适的类别文件中；增强型放 `enhanced.py`，深度财务放 `deep_finance.py`。
5. 写完后运行 `python build_factors.py --factor <name> --sequential`。
6. 检查 `data/manifests/<name>.json` 的覆盖率和非空行数。
7. 用 `ic_analysis.py` 的 `compute_ic(factor_name, target_name)` 验证因子 IC，至少横跨 1d/5d/10d 三个标签。
8. 财务数据因子耗时较长（~90s/个），开发时建议 `--sequential` 逐个构建调试；批量构建优先用 `--all --jobs 4` 并行。
9. 目前最快的因子类型是 price/valuation/event（<30s），最慢的是 financial（~90s）和 enhanced 中的 `price_volume_corr_20`（~11min）。
10. 因子总数已达 101 个，新增因子的边际价值递减。当前应优先做行业中性化、因子筛选、去极值，而非继续铺量。
